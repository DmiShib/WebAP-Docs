브라우저에서 최대 FPS를 끌어내세요. 지능적인 동적 스케일링과 정확한 병목(CPU/GPU) 분석이 게임을 모든 디바이스에 자동으로 최적화합니다.

동적 스케일링 (Dynamic Scaling)

GPU 부하를 줄이세요. 최대 FPS를 위한 네이티브 브라우저 스케일링을 선택하거나 선명한 UI를 유지하기 위한 스마트 렌더링을 선택하세요.

플러그 앤 플레이

프로젝트 설정(Project Settings)을 통해 두 단계로 구성됩니다. 프리팹을 씬에 배치할 필요 없이 코어 모듈이 빌드에 자동으로 주입됩니다.

12개의 최적화 모듈

그림자부터 LOD, 물리 엔진까지. 매끄러운 게임 플레이를 유지하기 위해 게임이 무엇을 먼저 희생할지 완전히 제어할 수 있습니다.

CPU와 GPU의 구분

무분별한 그래픽 품질 저하가 없습니다. 플러그인은 병목 현상(Bottleneck)을 정확하게 식별합니다: CPU 과부하 시 물리를 줄이거나 GPU 부족 시 그림자를 줄입니다.

안티 요요 페널티 시스템

지능형 페널티 록아웃(Penalty Lockout)이 품질 깜박임을 차단하여 부하 변동 중에도 안정적인 결과를 보장합니다.

Zero-GC 아키텍처

엄격한 메모리 절약: O(1) 원형 버퍼 및 Update에서의 메모리 할당 제로. 플러그인은 가비지 컬렉터(GC)를 유발하지 않으며 끊김 현상을 방지합니다.

개발자를 위한 공개 API ​

기성 에셋을 구매할 때 핵심 메커니즘에 접근하는 것은 필수적입니다. WebAP는 명확한 공개 API를 제공합니다. 스크립트를 통해 지표를 읽거나, 그래픽 품질을 고정하거나, 사용자 정의 최적화 모듈을 통합하세요.

단 몇 분 만에 당신의 고유한 메커니즘을 위한 커스텀 Scaler를 생성하세요:

csharp
using GrindsetStudios.WebAdaptivePerformance.Scalers;

[UnityEngine.Scripting.Preserve]
public sealed class MyCustomScaler : WebScalerBase
{
    public MyCustomScaler()
    {
        _target = ScalerTarget.GPU;                // 비디오 카드를 절약함을 명시
        _visualImpact = ScalerVisualImpact.Medium; // 시각적 결과에 미치는 영향 정도
        _maxLevel = 3;                             // 3단계의 품질 저하 단계 추가
    }

    protected override void OnLevel()
    {
        // 여기에 품질 설정을 적용하세요.
        // 플러그인이 언제 이 메서드를 호출할지 결정합니다!
    }
}

FPS를 떨어뜨려 보세요.
